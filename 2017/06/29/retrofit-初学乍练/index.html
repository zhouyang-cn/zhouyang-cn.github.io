retrofit-初学乍练

发表于 2017-06-29 | | 阅读次数

本文是一篇retrofit的入门文章，将会学习一下retrofit的配置及基础用法。

retrofit概述

先引用一下官方网站的介绍

A type-safe HTTP client for Android and Java. 适用于Android和Java的类型安全的HTTP客户端

retrofit官网：http://square.github.io/retrofit/

github库：https://github.com/square/retrofit

retrofit使用注解来描述http请求，默认情况下集成了URL参数替换和查询参数支持。此外，还提供自定义headers，multipart request，文件上传和下载，mock响应结果等等。

在android项目中使用retrofit

官方建议我们使用的是retrofit2，因为retrofit2利用OKHttp作为默认网络层，不需要我们明确定义OKHttp作为你的项目依赖，除非你需要特定版本的OKHttp.

定义依赖

由于retrofit2需要依赖一个converter来自动转换Request和Response，所以我们还需要导入一个converter库，我这里使用的是gson的converter库。

网络权限声明

由于我们要发起网络请求，所以要在AndroidManifest.xml文件中声明网络权限。

使用retrofit发起网络请求

在发起请求之前，你需要描述你想要与之交互的API的路径(Endpoints)。本篇文章我们将使用GitHub的api来进行网络请求。（如果不了解RESTful API 的话，可以先看一下阮一峰老师的博客。）
